## 2. 시장 리서치 핵심 발견

### 2.1 사용자가 가장 원하는 것 (우선순위순)

1. **RFC 9457 Problem Details** — Stripe, GitHub, Cloudflare가 채택한 표준 에러 포맷. NestJS에서 이를 응답 래핑과 통합한 라이브러리 없음
2. **HATEOAS 링크** — AI 에이전트 시대에 자기 설명적 API 재부상. 페이지네이션에 `links` 필드 요구
3. **응답 시간 메타데이터** — `meta.responseTime` (ms) 으로 디버깅/모니터링 지원
4. **정렬/필터링 메타데이터** — 현재 어떤 정렬/필터가 적용됐는지 응답에 반영
5. **SDK 생성 친화적 OpenAPI** — openapi-generator로 깔끔한 타입 생성되는 스키마

### 2.2 경쟁사가 못 하는 것

- Swagger + 제네릭 래퍼 = 문서가 깨짐 (우리는 이미 해결)
- 성공 래핑 OR 에러 포맷팅 중 하나만 제공 (우리는 둘 다)
- ORM에 종속된 페이지네이션 (우리는 ORM 무관)
- Express 전용 (우리는 Fastify도 지원)

### 2.3 경쟁 환경

| 라이브러리 | Swagger | 페이지네이션 | 에러 처리 | Fastify |
|-----------|---------|------------|----------|---------|
| nest-standard-response | 커스텀 데코레이터 | 오프셋만 | 없음 | 불확실 |
| @zabih-dev/nest-api-response | 기본 | 없음 | 기본 | 불확실 |
| nestjs-paginate | 부분 | 오프셋+커서 | 없음 | 지원 |
| nest-problem-details | 없음 | 없음 | RFC 7807만 | 지원 |
| **nestjs-safe-response** | **완전 통합** | **오프셋+커서** | **코드 매핑** | **지원** |

> 시장은 파편화되어 있음. Swagger 문서 + 응답 엔벨로프 + 페이지네이션 + RFC 9457 에러 + 듀얼 어댑터를 모두 제공하는 라이브러리가 없음.

## 3. v0.8.0 제안: 3개 테마

### 테마 A: RFC 9457 Problem Details (에러 응답 고도화)

**영향도: 높음 | 난이도: 중**

현재 에러 응답:
```json
{
  "success": false,
  "statusCode": 404,
  "error": { "code": "NOT_FOUND", "message": "User not found" }
}
```

RFC 9457 호환 모드 추가:
```json
{
  "type": "https://api.example.com/problems/not-found",
  "title": "Not Found",
  "status": 404,
  "detail": "User with ID 123 not found",
  "instance": "/api/users/123",
  "code": "NOT_FOUND",
  "requestId": "abc-123"
}
```

#### 구현 항목

| 항목 | 설명 |
|------|------|
| `problemDetails` 모듈 옵션 | `true`로 설정하면 에러 응답이 RFC 9457 포맷 |
| `Content-Type: application/problem+json` | 에러 시 자동 헤더 설정 |
| `@ProblemType(uri)` 데코레이터 | 라우트별 `type` URI 지정 |
| `ProblemDetailsDto` Swagger DTO | OpenAPI에 RFC 9457 스키마 자동 문서화 |
| `@ApiSafeProblemResponse()` 데코레이터 | Problem Details 전용 Swagger 데코레이터 |
| 기존 포맷과 호환 유지 | `problemDetails: false`(기본값)이면 현재 동작 그대로 |

#### 차별화 포인트

- [nest-problem-details](https://github.com/Fcmam5/nest-problem-details)는 에러만 처리하고 성공 래핑 없음
- 우리는 성공 래핑 + RFC 9457 에러를 하나의 모듈에서 통합 제공
- Swagger 문서에 `application/problem+json` 스키마 자동 생성

### 테마 B: 페이지네이션 메타데이터 확장 (Links + Sorting)

**영향도: 중 | 난이도: 중**

#### B-1. HATEOAS 페이지네이션 링크

```json
{
  "meta": {
    "pagination": {
      "type": "offset",
      "page": 2,
      "limit": 20,
      "total": 100,
      "links": {
        "self": "/api/users?page=2&limit=20",
        "first": "/api/users?page=1&limit=20",
        "prev": "/api/users?page=1&limit=20",
        "next": "/api/users?page=3&limit=20",
        "last": "/api/users?page=5&limit=20"
      }
    }
  }
}
```

커서 페이지네이션 링크:
```json
{
  "meta": {
    "pagination": {
      "type": "cursor",
      "links": {
        "self": "/api/feed?limit=20",
        "next": "/api/feed?cursor=eyJ...&limit=20",
        "prev": "/api/feed?cursor=eyJ...&limit=20"
      }
    }
  }
}
```

| 항목 | 설명 |
|------|------|
| `PaginatedOptions.links` | `true`로 설정하면 링크 자동 생성 |
| `CursorPaginatedOptions.links` | 커서 페이지네이션에도 링크 지원 |
| `PaginationLinksDto` Swagger DTO | 링크 스키마 문서화 |
| 링크 자동 생성 로직 | `request.url` 기반 쿼리 파라미터 조합 |

#### B-2. 정렬/필터 메타데이터 (선택적)

```json
{
  "meta": {
    "pagination": { "..." },
    "sort": { "field": "createdAt", "order": "desc" },
    "filters": { "status": "active", "role": "admin" }
  }
}
```

| 항목 | 설명 |
|------|------|
| `@SortMeta()` 데코레이터 | 정렬 정보를 meta에 포함 |
| `@FilterMeta()` 데코레이터 | 적용된 필터를 meta에 포함 |
| `SortMetaDto`, `FilterMetaDto` | Swagger 스키마 자동 생성 |

### 테마 C: DX 및 품질 개선

**영향도: 중 | 난이도: 하~중**

| 항목 | 설명 | 난이도 |
|------|------|--------|
| 응답 시간 메타데이터 | `meta.responseTime` (ms) 자동 계산 옵션 | 하 |
| `@ApiSafeResponse` 다중 상태코드 | 하나의 엔드포인트에서 200/201 모두 문서화 | 하 |

| `@nestjs/swagger` v11 완전 호환 | `operationId`, `tags` 등 v11 신기능 연동 확인 | 하 |
| ESM 지원 | `"type": "module"` 이중 빌드 (CJS + ESM) | 중 |

## 6. API 설계 프리뷰

### 6.1 RFC 9457 활성화

```typescript
@Module({
  imports: [
    SafeResponseModule.register({
      problemDetails: true,  // RFC 9457 모드 활성화
      problemBaseUrl: 'https://api.example.com/problems',
    }),
  ],
})
export class AppModule {}
```

### 6.2 Problem Details 커스텀

```typescript
@Controller('users')
export class UsersController {
  @Get(':id')
  @ProblemType('user-not-found')  // → type: https://api.example.com/problems/user-not-found
  @ApiSafeProblemResponse(404, {
    title: 'User Not Found',
    detail: 'The requested user does not exist',
  })
  async findOne(@Param('id') id: string) {
    // ...
  }
}
```

### 6.3 HATEOAS 링크

```typescript
@Controller('users')
export class UsersController {
  @Get()
  @Paginated({ maxLimit: 100, links: true })  // links 옵션 추가
  @ApiPaginatedSafeResponse(UserDto)
  async findAll(@Query('page') page = 1, @Query('limit') limit = 20) {
    return { data: users, total, page, limit };
  }
}
```

### 6.4 응답 시간

```typescript
SafeResponseModule.register({
  responseTime: true,  // meta.responseTime (ms) 자동 포함
})
```

응답 예시:
```json
{
  "success": true,
  "statusCode": 200,
  "data": { "..." },
  "meta": {
    "responseTime": 42
  }
}
```

## 7. 비호환 변경 (Breaking Changes)

v0.8.0에서는 **비호환 변경 없음** 을 목표로 합니다.

- RFC 9457은 `problemDetails: false`(기본값)으로 기존 동작 유지
- HATEOAS 링크는 `links: false`(기본값)으로 기존 동작 유지
- 응답 시간은 `responseTime: false`(기본값)으로 기존 동작 유지
- 모든 신규 기능은 opt-in 방식
